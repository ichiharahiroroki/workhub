# リソース管理の考え方

WorkHub のリソース管理は、チケットの見積工数、担当者の予想戦闘力、投下工数を突き合わせて、計画が現実的かどうかを判断するための考え方である。

見積工数は一度決めて終わりではなく、実績との差分を振り返りながら、チケット分解、見積工数、戦闘力を継続的に調整する。

## 🧮 見積工数の扱い

タスクの見積工数は、基準実力を持つ人がそのタスクを実行した場合に、どれくらいの時間で完了できそうかを表す。WorkHub では、[見積もり基準](./estimation-criteria.md) に基づき、AI がタスクチケットごとに初期見積もりを行う。

戦闘力は、1日（8時間）で消化できる見積工数を表す。基準となる実力を持った人が1日に消化できる工数を `基準実力` とし、担当者ごとの実力差を表す `潜在能力` を掛けて算出する。

```text
戦闘力 = 基準実力 × 潜在能力
```

| 要素 | 説明 |
| --- | --- |
| 基準実力 | 基準となる実力を持った人が1日に消化できる見積工数 |
| 潜在能力 | 基準実力に対する担当者ごとの相対的な実力 |

成果物、グループ、マイルストーンなど、配下タスクを持つチケットの見積工数は、配下タスクの見積工数の合計で算出する。

```text
上位チケットの見積工数 = 配下タスクの見積工数の合計
```

## ⚡ 予想戦闘力の計算

予想戦闘力とは、担当者が対象期間に実際に作業へ使える作業可能量を表す指標である。WorkHub では、営業日、非作業日、カレンダー上の拘束予定、1日の稼働時間、潜在能力をもとに計算する。

```text
予想戦闘力 =（営業日 - 非作業日 - Reserved）× 1日の稼働時間 × 潜在能力
```

| 要素 | 説明 |
| --- | --- |
| 営業日 | 対象期間の営業日数 |
| 非作業日 | 有給・祝日など |
| Reserved | Calendar 上の拘束予定（会議、定例、移動等） |
| 1日の稼働時間 | 1日に作業へ使える基準時間（hour） |
| 潜在能力 | 4年目エンジニア = 1.0 を基準とした相対能力 |

## 🔁 振り返りで見るもの

振り返りでは、見積もりと実績の差分を確認し、次回のチケット分解や見積もりに反映する。

| 項目 | 説明 |
| --- | --- |
| 投下工数 | タスクの実行に実際に使った時間の合計 |
| 実戦闘力 | 実際の投下工数で、どれだけの見積工数を消化できたかを1日換算したもの |

```text
実戦闘力 = 消化した見積工数 ÷ 投下工数 × 1日の稼働時間
```

予想戦闘力と実戦闘力の差分は、単純に大きければよく小さければ悪いものではない。乖離の理由を振り返り、潜在能力やチケットの見積工数を見直す。
