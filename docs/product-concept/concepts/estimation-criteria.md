# 見積もり基準

このドキュメントは、WorkHub でタスクチケットの見積工数を出すときの基準を定義する。実際の見積もりは AI の skill で行うが、その skill が参照する判断基準をここにまとめる。

## 🧑‍💻 基準となる人

基準となる人とは、対象領域の基本的な知識と実装経験を持ち、既存の設計やコードを読みながら、標準的な品質でタスクを完了できる人を指す。

WorkHub では、現時点では 4年目エンジニア相当を基準の目安にする。ここでの基準は、特定個人の能力ではなく、タスクの見積工数を揃えるための共通の物差しである。

## 🧮 見積工数の考え方

見積工数は、基準となる人がそのタスクを実行した場合に必要になりそうな作業時間である。担当者ごとの実力差は見積工数には含めず、潜在能力や戦闘力の計算で扱う。

難易度や得意不得意を別係数として持つのではなく、必要な調査、実装、確認、レビュー対応などを含めて、基準となる人なら何時間で終わりそうかを見積もる。

## 🔍 AI が見る情報

AI は、タスクチケットの説明、目的、背景、完了条件、関連チケット、過去の判断履歴、対象画面や対象ファイル、想定される確認方法をもとに初期見積もりを行う。

見積もりに必要な情報が不足している場合は、無理に確定せず、不足している前提や確認事項を明示する。

## 🛠️ 人間が補正する観点

AI の初期見積もりは、最終的な確定値ではない。人間は、実装上の制約、既存コードの状態、外部依存、レビュー方針、チケット分解の粒度を踏まえて、必要に応じて見積工数を調整する。

見積工数を調整した場合は、なぜ調整したのかをチケットの説明や変更履歴に残す。これにより、次回以降の見積もりや振り返りに使える判断材料を蓄積する。

## 📦 skill に持たせる前提

見積もり skill では、このドキュメントの基準に沿って、タスクチケットごとに見積工数を出す。

skill は少なくとも、以下を出力できるようにする。

| 出力 | 内容 |
| --- | --- |
| 見積工数 | 基準となる人が実行した場合に必要な時間 |
| 見積根拠 | どの情報をもとに見積もったか |
| 不足情報 | 見積もりの確度を下げている未確認事項 |
| 分解提案 | タスクが大きすぎる場合の分割案 |
