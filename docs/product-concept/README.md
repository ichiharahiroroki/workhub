# WorkHub タスクマネジメント・プレイブック

## 🌐 はじめに

**WorkHub は、社内のデジタル化プロジェクトに関わる人間、AI、外部ツール、コンテキストを一元的に紐付け、人間とAIが同じ文脈を共有しながら協働できるHubを目指す。**

本ドキュメントでは、WorkHub を理解するための核となる考え方を定義する。

---

<a id="core-concepts"></a>

## 🧠 Core Concepts（核となる考え方）

### 🎫 チケット制を取り入れてすべてを紐付ける

WorkHub は、仕事を [チケット制](./concepts/ticket-management-system.md) で扱う。

社内のデジタル化プロジェクトでは、開発タスク、調査、資料作成、会議準備、突発依頼、学習・検証タスクなど、多様なタスクが同時に発生し、GitHub、Notion、Google Drive、Slack、Calendar など複数のツールに分散しやすい。WorkHub ではそれらをチケット単位で扱い、内容、ステータス、完了条件などの基本情報だけでなく、マイルストーン、成果物、GitHub Issue / PR、Notion ページ、Google Drive ファイル、Slack スレッド、AI セッションなどの関連情報も紐付けて追跡できる。

特に、デジタル化や効率化を目的とするタスクやプロジェクトでは、開発時の判断理由や背景が、長期的な保守・改善・エンハンス時に重要な参照情報になる。だからこそ、管理自体もデジタル化し、判断や経緯を個人に閉じさせず、後から追える状態にしておく必要がある。

### 🤖 人間とAIのコラボレーションを支える

WorkHub では、人間とAIが同じチケット情報を参照して協働する。AI はチケットに紐付いた目的、背景、関連資料、過去の判断履歴、完了条件などを理解して、タスク遂行の支援を行うことができる。

タスクの目的、背景、判断理由、完了条件などをチケットに集約することで、依頼人、作業者、AI の間で認識を合わせやすくなる。AI に作業を依頼する場合も、背景や判断履歴、参照資料を毎回説明し直す手間を減らせる。

### ✅ チケットごとにレビューしやすくする

WorkHub では、タスクをチケット単位に細分化して管理するため、レビュー対象とレビュー観点を整理しやすくなる。また、このツール上でレビューができる。

### 🗂️ 管理区分でさまざまな仕事を扱う

WorkHub では、[管理区分](./concepts/work-management-categories.md) を使って、性質の異なる仕事を同じチケット管理の枠組みで扱う。

| 管理区分 | 扱う仕事 | 補足 |
| --- | --- | --- |
| Project | ゴールや成果物に向けて計画的に進める仕事 | スプリント単位で計画・実行・振り返りを行う |
| Ad-hoc | 突発依頼や単発対応 | Project に属さない仕事を扱う |
| Explore | 学習・検証・実験のための仕事 | 定量化しにくい探究活動も見積もり・実績を追跡する |
| Reserved | カレンダー上の拘束時間 | タスク管理の対象ではなく、実行可能性の判断に使う |

### 🧮 リソース管理で計画の現実性を判断する

WorkHub では、[リソース管理の考え方](./concepts/resource-management.md) を定義し、チケットの見積工数、担当者の作業可能量、投下工数を突き合わせて計画の現実性を判断する。タスクの見積工数は [見積もり基準](./concepts/estimation-criteria.md) に基づいて扱い、実績との差分を振り返ることで、次の計画やチケット分解の精度を高める。

### 📈 リソース情報で実行可能性を判断する

WorkHub は、各チケットの見積工数、投下工数、進捗、担当者、期限と、担当者の稼働可能時間を突き合わせて、成果物やマイルストーンの達成時期を予測することが可能。

タスクが増えても、何を優先すべきか、どこまで受けられるか、誰が対応可能かを判断しやすくなり、過剰なコミットメントによる計画の破綻を防ぎやすくなる。
