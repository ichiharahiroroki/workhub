# Beyond R UI プロトタイプ 認識合わせメモ

このページは、Beyond R の React UI プロトタイプを作るための作業共有ページです。
親エージェントはここを更新しながら、ユーザーとの認識合わせ・サブエージェントの担当整理・レイアウト意図の共有を行います。

## 目的

- 社内提案時に、Beyond R がどのようなツールになるかを視覚的に伝える。
- 将来的に、ツールを使い始める人向けのチュートリアル画面としても流用できる。
- DB・認証・外部API・バックエンドスクリプトは作らず、React の静的プロトタイプとして使用感を表現する。

## 親エージェントの現時点のUIイメージ

Beyond R は「タスク一覧」ではなく「使えるリソースの総量」から判断するツールなので、最初の画面はダッシュボード型にします。
画面左にナビゲーション、上部にスプリントと期間、中央にリソース残量・予想戦闘力・実戦闘力・配分状況を置きます。

メインビューは次の3つを切り替えられる想定です。

1. Resource Board: 今どれだけ余裕があるか、Project / Ad-hoc / Explore / Reserved にどう配分されているかを見る画面
2. Drilldown: Project からマイルストーン、成果物、タスクへ掘り下げる画面
3. Tutorial: 初めて使う人に「何を見るべきか」を説明するオンボーディング画面

## サブエージェント担当

- 親エージェント: React/Vite の土台、共有データ、画面構成、最終統合、起動確認
- Agent A: 依存関係が切れた後に、Resource Board 用のカード・表現部品を量産
- Agent B: 依存関係が切れた後に、Tutorial / Onboarding 用の説明ステップ部品を量産
- Agent C: 必要になった場合のみ、空状態・サンプルデータ・細かな表示文言などの独立タスクを担当

- 実装範囲: 静的UIのみ。DB、認証、外部API、バックエンドスクリプトは作らない。

  - Tutorial: 初回利用者向けに、見る順番と判断方法を案内する
- トーン: 社内提案に使える、落ち着いた業務ツール。派手なLPではなく、実際に使う画面を最初に見せる。

### ドリルダウンの認識合わせ

現時点で親エージェントが理解している「ドリルダウン」は、単に左メニューをクリックして右側が変わることではない。

- Project の場合: `Projects -> Project名 -> マイルストーン -> 成果物 -> タスク`
- Ad-hoc の場合: `Ad-hoc -> カテゴリ -> 成果物 -> タスク`
- Explore の場合: `Explore -> カテゴリ -> 成果物 -> タスク`
- 上位階層を選ぶと、その直下の子要素が一覧で表示される
- 子要素をクリックすると、さらに下位階層へ進む
- パンくずで現在位置が分かる
- 選択中ノードの詳細と、子要素一覧が同時に見える
- リソース画面に表示されるタスクは、このドリルダウン階層のどこかに必ず存在する

確認したいこと:

- Flagxs に近づける場合、WBSは「表形式で階層行を展開する」イメージか、それとも「左から右へカラムで掘る」イメージか。
- Project / Milestone / 成果物 / タスクのうち、ユーザーが日常的に新規作成するのはどの階層か。
